# PRD — python期末實作（校園附近美食地圖｜國立宜蘭大學）

## 1. 問題與目標

- **問題陳述**：宜大學生在校外覓食時，難以在「距離可接受」範圍內，依網路評價（星等＋評論量）快速排序、篩選餐廳。
- **目標使用者**：國立宜蘭大學學生（以步行／單車可達範圍為主）。
- **校園中心**：宜大校本部（神農路一段1號）約 `24.7464°N, 121.7457°E`。
- **成功條件**：
  - 可設定校園中心座標與搜尋半徑，列出半徑內餐廳。
  - 排序同時考量「黃氏星等」（星等分級＋星分權重＋評論量）與「距離」（綜合分數）；另提供貝氏星等排序供比較。
  - 提供 CLI 與 Streamlit 網頁兩種操作方式；資料可離線（內建 Mock JSON），方便期末 demo。

## 2. 功能邊界（In Scope）

- **FR-1**：依中心點緯經度、半徑（公里）篩選餐廳。
- **FR-2**：依最少評論數篩選；依「綜合／貝氏星等／黃氏星等／距離」四種模式排序。
- **FR-3**：輸出含 Google Maps 搜尋連結（以座標開啟）；CLI 支援 `table`／`json`。
- **FR-4**：Streamlit 顯示表格與簡易地圖點位；校園中心預設宜大校本部，可由側欄或環境變數 `CAMPUS_LAT`／`CAMPUS_LON` 覆寫。
- **FR-5**：可選 `--data`／側欄路徑載入自訂餐廳 JSON（陣列格式，欄位見 `sample_restaurants.json`）。

## 3. 非目標（Out of Scope）

- **NFR-1**：本階段不接 Google Places／Maps Platform 付費 API（僅 Mock／自建 JSON）。
- **NFR-2**：不實作帳號、收藏、即時營業狀態、導航路線規劃。

## 4. KPI（可量測，同步 `slo.config.json`）

- `p95_latency_ms <= 120.0`
- `timeout_rate <= 0.002`
- `cost_per_request <= 0.2`
- `quality_drop_ratio <= 0.05`
- `error_rate <= 0.01`
- 相對門檻：p95 ≤ baseline×0.75；cost ≤ baseline×0.7

## 5. NO-GO 條件

- 任一核心測試失敗
- 任一 KPI 超標
- 觀測資料缺失（artifact 不完整）

## 6. 風險與假設

- **風險**：Mock 資料（範例命名＋虛構評分）與真實商圈落差；Streamlit 部署需額外依賴。
- **假設**：宜大校本部座標為示範用；JSON 欄位格式正確。
- **緩解策略**：範例餐廳採「街道＋品項」命名以**避免冒用真實店家評分**；後續可加 `GoogleMapsProvider` 不改核心排序介面。
